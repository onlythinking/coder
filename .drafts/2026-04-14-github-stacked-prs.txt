## Reddit 草稿（r/programming）

**Title**: GitHub launched Stacked PRs (Private Preview) - a systematic approach to reviewing large code changes

**Body**:
After years of big tech using internal tools like Google's Piper and Meta's Stacked Diff to manage large code changes, GitHub has finally brought this workflow to open source.

Stacked PRs lets you break a large refactor (say, 3000+ lines) into a chain of small, focused PRs that build on each other. When the base PR merges, all downstream PRs automatically update their base branch.

Key benefits:
- Reviewers see smaller, focused diffs
- No manual rebase when upstream merges
- Parallel review without blocking
- Per-layer rollback

The `gh-stack` CLI extension manages the entire workflow:
```bash
gh extension install github/gh-stack
gh stack init feature/layer-1
gh stack add feature/layer-2  # auto-bases on layer-1
gh stack push
```
